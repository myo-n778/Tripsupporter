# TripSupporter current-spec

## 目的

修学旅行・校外学習向けに、班ごとの観光ルートと時刻表を作成し、Google Apps Script 経由でスプレッドシートへ保存するウェブアプリとして動作させる。

## 現行仕様

- React + Vite の単一ページアプリとして起動する。
- 生徒は班番号 `01` から `12` と PIN でログインする。
- 管理者は班番号 `admin`、PIN `admin` でログインする。
- 管理者画面から、各班が各時刻にどこに滞在しているかを示すガントチャート型タイムラインをスプレッドシートへ出力できる。
- タイムライン出力先は同一スプレッドシート内の `gid=1348031489` のタブとする。
- タイムラインは、2日目・3日目ごとに班番号を縦軸、時刻を横軸として表示する。
- タイムラインは15分刻みのセルを横方向に並べ、滞在時間に合わせてセルを結合した横棒として場所名を表示する。
- タイムラインでは、通常の滞在場所は緑、昼食・カフェ等はオレンジで表示する。
- タイムラインでは、移動時間は横棒にせず空白で表示する。
- タイムラインには説明文の長文行を出さず、表本体を優先して表示する。
- 滞在時間0分の通過地点は、同じ時刻枠に次の滞在地点がある場合、次の横棒のラベルへ `通過地点 → 滞在地点` の形で併記して表示する。
- 滞在時間0分の通過地点が行程末尾にある場合は、最小1枠の横棒として表示する。
- タイムライン生成時は、保存データに到着時刻・出発時刻がある場合、それを優先してUI上の時刻と揃える。
- 生徒画面では 2 日目、3 日目の行程を作成する。
- 出発地、目的地、移動手段、移動時間、滞在時間を入力できる。
- 駅や乗換地点などの通過地点では、滞在時間0分を入力できる。
- 行程表では到着時刻、出発時刻を自動計算する。
- 行程表の各地点には、地図上のマーカーと同じ番号を表示する。
- 保存時は GAS URL へ POST する。
- 読み込み時は GAS URL から GET する。
- Google Maps API キーが設定されている場合、目的地に基づいて地図とルートを表示する。
- 地図には登録済みの出発地・目的地を番号付きマーカーとして表示する。
- 地点候補から選択した場所は `placeId` を保持し、地図表示時はその場所を優先して特定する。
- 手入力地点が明確に1件へ特定できない場合は、誤った地点を示さないため地図には表示しない。
- 地図には登録済みの出発地・目的地を赤い円でも強調表示する。
- 地点の赤い強調表示は、ズームに左右されにくいリング型マーカーも併用する。
- 目的地間は各区間の移動手段に応じて徒歩または公共交通の経路線を描画する。
- 経路線は区間ごとに色を変え、行程表の移動手段表示にも対応色を表示する。
- 地点追加・日付切替時は、入力済み地点が見える範囲へ地図を自動追尾する。
- 各地点カードのチェックボックスで、地図上の丸・マーカー・追尾対象を一時的に表示/非表示できる。
- 地図の自動追尾は、チェックが入っている全地点が映る範囲へ調整する。1地点だけの場合も寄りすぎない。
- 「見積り(徒歩)」ボタンは、徒歩移動の目安取得を主目的とする。
- 移動手段が電車またはバスの場合、移動手段欄の右側に赤字太字で「所要時間は検索してください」と表示する。
- 出発地、目的地の入力時にGoogle Places候補を表示し、候補から選択できる。
- 候補選択時に入力欄へ入れる文字は、住所全文ではなく `奈良駅` のような短い主名称にする。
- 京都・奈良限定モードをUIチェックボックスで切り替えられる。
- 京都・奈良限定モードでは、候補検索とジオコーディングを京都・奈良周辺へ寄せる。
- 乗車場所・降車場所欄は設けず、駅名や乗換地点も目的地として追加する。
- 公共交通の正確な経路が返らない場合、車や徒歩の時間では代用せず、手入力を促す。
- Places写真が取得できる地点は、行程カード右側に小さな写真プレビューを表示する。
- 写真プレビューはタップで拡大し、拡大表示を再タップすると戻る。

## 環境変数

- `VITE_GOOGLE_MAPS_API_KEY`: Google Maps JavaScript API 用キー。
- `VITE_GAS_URL`: Google Apps Script のウェブアプリ URL。
- `VITE_GAS_PROXY_URL`: ローカル開発時に GAS へ中継する Vite proxy のパス。既定は `/gas`。
- `VITE_ADMIN_PIN`: 管理者ログイン用PIN。班番号 `admin` と組み合わせて使う。
- 電車で見積もる場合はGoogle Mapsの公共交通モードに `TRAIN` を、バスで見積もる場合は `BUS` を明示する。

## 変更対象

- Vite/React として起動するためのプロジェクト構成。
- アプリ本体 `src/App.jsx`。

## 変更しないもの

- GAS 側の既存保存・読込仕様。
- スプレッドシートの構造。
- Google Cloud Console 側の API 設定。
- 班番号、PIN、2 日目/3 日目という基本運用。
