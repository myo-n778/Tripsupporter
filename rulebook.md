# TripSupporter rulebook

## 優先順位

1. `current-spec.md`
2. 実際に動作しているアプリ本体
3. この変更履歴
4. 過去メモや会話

## 変更ルール

- 正常動作しているログイン、保存、日付切替、行程計算は目的なく変更しない。
- Google Maps API キーや GAS URL はソースへ直書きせず、環境変数で扱う。
- ローカル開発時の GAS 通信は CORS と GAS リダイレクト回避のため Vite の開発用中継を使う。
- GAS 通信の仕様を変える場合は、GAS 側とスプレッドシート構造への影響を先に確認する。
- 管理者画面と生徒画面は分けて回帰確認する。
