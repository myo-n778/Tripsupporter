# TripSupporter regression-checklist

## 起動

- `npm run dev` で開発サーバーが起動する。
- ログイン画面が表示される。
- Tailwind のスタイルが反映される。
- ローカル開発時は `/gas` 経由で GAS にアクセスできる。

## 生徒画面

- 班番号と PIN を入力できる。
- 班番号は `01` から `14` の選択式で入力できる。
- スプレッドシート側のPINが数値でも文字列でもログイン判定できる。
- 2 日目、3 日目を切り替えられる。
- 出発地を追加できる。
- 目的地を追加できる。
- 移動時間と滞在時間から到着・出発時刻が計算される。
- 滞在時間0分の目的地を追加できる。
- 目的地を上下移動できる。
- 目的地を削除できる。
- 保存ボタンが GAS へ POST する。

## 管理者画面

- `admin` / `admin` で管理者画面に入れる。
- 全班データが表示される。
- 2 日目、3 日目を切り替えられる。
- データ最新化が GAS へ GET する。
- タイムライン出力ボタンが表示される。
- タイムライン出力中は出力中メッセージとボタンの `出力中...` 表示が出る。
- タイムライン出力は GAS の `action=timeline` を呼び出す。
- GASが古いデプロイで `action=timeline` を処理できない場合、具体的な更新案内が表示される。
- タイムライン出力先は `gid=1348031489` のタブである。
- タイムラインには 2日目・3日目ごとに `01` から `14` 班の行が表示される。
- タイムラインには時刻列が15分刻みで横方向に表示される。
- 滞在場所は滞在時間に応じて横方向にセル結合された色付きバーで表示される。
- 通常の滞在場所は緑、昼食・カフェ等はオレンジで表示される。
- 移動時間はバー表示せず空白になる。
- タイムラインに長い説明文行は表示されない。
- 滞在時間0分の通過地点も最小1枠で表示される。
- 保存済みの到着時刻・出発時刻がある場合、タイムラインはその時刻を優先する。

## 地図

- API キー未設定時は準備中表示になる。
- API キー設定時は Google Maps スクリプトが読み込まれる。
- ログイン前に Google Maps スクリプトが読み込み済みでも、ログイン後に地図が初期化される。
- ログアウトや管理者画面遷移後に生徒画面へ戻っても、古い地図インスタンスを使い回さない。
- 目的地が 2 件以上あるとルート描画を試みる。
- Google Maps APIキーが拒否された場合、確認すべき設定がエラー表示に出る。
- 目的地が 1 件でも番号付きマーカーが表示される。
- 目的地が複数ある場合、各地点に番号付きマーカーが表示される。
- 行程表の各地点にも地図と同じ番号が表示される。
- 候補から選んだ地点は `placeId` を使って地図上の場所を特定する。
- 手入力地点が明確に1件へ特定できない場合、地図上の丸・マーカー・ルートには出さない。
- 各地点に赤い円の強調表示が出る。
- 各地点にズームに左右されにくい赤いリング表示が出る。
- 地点追加や日付切替後、地図が地点範囲へ自動で移動・ズームする。
- 地図は最後の地点だけではなく、表示中の全地点が映るように調整される。
- 各地点カードのチェックを外すと、その地点の丸・マーカー・ルートが地図から消える。
- チェックを外した地点は地図の追尾範囲から外れる。
- 地図表示チェックのON/OFFは保存後の再読み込みでも維持される。
- 1地点だけ表示する場合も過度にズームしすぎない。
- 目的地間は区間ごとの移動手段に応じて経路線が描画される。
- 複数区間の経路線は区間ごとに異なる色で描画される。
- 行程表の移動手段チップに、地図上の経路線と対応する色が表示される。
- ボタン表示は `見積り(徒歩)` である。
- 移動手段が電車またはバスの場合、移動手段欄の右側に赤字太字で `所要時間は検索してください` が表示される。
- 移動手段が徒歩の場合、`所要時間は検索してください` は表示されない。
- 徒歩移動ではGoogle Mapsの徒歩経路時間が移動時間欄に反映される。
- 乗車場所・降車場所欄は表示されない。
- 出発地、目的地の入力時に候補リストが表示される。
- 候補を選ぶと入力欄へ反映される。
- 候補を選んだとき、入力欄には住所全文ではなく短い主名称が入る。
- 京都・奈良限定チェックをオンにすると、候補と地点検索が京都・奈良周辺へ寄る。
- 公共交通の見積りが `ZERO_RESULTS` などで失敗した場合、車の時間を入れずに手入力を促す。
- 電車見積りは `TRAIN`、バス見積りは `BUS` を指定してGoogle Mapsへ問い合わせる。
- 取得できる地点写真がある場合、行程カード右側にプレビューが表示される。
- 写真プレビューをタップすると拡大し、再タップで閉じる。
- 写真が取得できない地点でも行程カード表示は崩れない。

## PIN確認

- 管理者画面で各班カードにPINが表示される。
